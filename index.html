「Webコンテンツ作成」の成果

本テーマについて

このテーマでは、HTML,CSS,JavaScriptの基本を学び、これらを組み合わせてWebページを構成する方法を学びます。私はこのテーマで、Webページの土台を作り、ある程度自由にWebページを構成できるようになりたい。

プロジェクト課題について

私はこのプロジェクトで、ホットペッパーのグルメ情報を選択し、ユーザーがジャンルのキーワードを入力すると該当する飲食店の情報が表示されるWebページを作成しました。Googleの検索結果ページを参考にし、情報が読みやすく整っていて、使いやすいレイアウトを意識しました。HTMLとCSSで基本的な構造やデザインを整え、JavaScriptによって検索処理と動的な表示を実現しました。見た目のシンプルさと情報のわかりやすさを両立させるよう工夫しながら制作しました。

  • プロジェクト課題で作成したページ：プロジェクト課題 HTML

  • プロジェクト課題で作成したページ：プロジェクト課題 CSS

数当てゲーム

第3回の講義では、JavaScriptを使って1〜10の中からランダムに選ばれる数字を当てる「数当てゲーム」を作成しました。まずはプログラミングIで学んだ数値の扱いや乱数の生成方法を用いて、if文を用いた条件分岐によって「正解」「大きすぎる」「小さすぎる」といったフィードバックを返す仕組みを作りました。この作業を通して、単純な処理の中にも多くのロジックが含まれていることに気づき、JavaScriptの面白さを実感しました。また第4回では、HTMLと組み合わせて、テキストボックスやボタンをページに配置し、ユーザーが実際に数値を入力してゲームに参加できるインターフェースを整えました。入力された値がリアルタイムで反映されることで、Webページが「ただ表示するもの」から「双方向にやり取りできるもの」へと変化したことが印象的でした。今回は基本的なゲームでしたが、ゆくゆくは「まるバツゲーム」や簡単なクイズなど、より複雑なルールを持つプログラムにも挑戦していきたいと思っています。

各授業回の学習事項

第1回：WWWとHTML

このテーマではHTML文章の構成を理解し、head,body要素やh1,p,aなどといった基本的なHTML要素を学んだ。実際にHTMLによってWebページを記述する方法について学んだ。

  • 演習1-3：HTML文書の構成
  • 演習1-4：HTMLのさまざまな要素

第2回：CSSによるWebページの装飾

このテーマではCSSを使って、HTML文章に構成したWebページに色や文字サイズ、余白などの装飾を加える方法について学んだ。見やすいwebページを作るためには欠かせない要素であると感じた。

  • 演習2-3：ブラウザ紹介ページの装飾

第3回：JavaScriptの基本

このテーマではJavaScriptを使って、webページ上でプログラムを動かす方法を学んだ。これが使えるのと使えないのではWeb上でできることが大きく変わると知ったため、頑張って習得したい。

  • 演習3-6：オブジェクトの参照

第4回：DOMとJavaScript

このテーマではHTML要素をJavaScriptで操作する方法（DOM操作）を学んだ。ボタンを用いることでユーザーからの入力でWebページの内容を変更したり、テキストボックスに数字を入れて演算をさせたりすることができるようになった。

  • 演習4-10：足し算

第5回：イベント処理とDOMの追加・削除

このテーマでは前回学んだDOM操作の他に、要素を新規に作成する方法や要素を編集する方法、そして要素の削除の方法について学んだ。これによって前回学んだボタンを押されたときに新たな要素(文章など)を表示させることができるようになった。

  • 演習5-7：キャンパスの住所表示

工夫点,苦心点,強調点,感想

プロジェクト課題ではこれまでの講義で学び、技術を身につけたHTML・CSS・JavaScriptを使って、グルメ情報を検索・表示するWebページを作成しました。実際に使えるデータ（HotPepperのグルメデータ？）を使って、自分のページでキーワード検索をして店舗情報を出せるようにするという内容で、これまで学んできた知識、技術を活かす実践的な課題でした。 特に苦労したのはキーワード検索がうまく機能しない問題です。検索ボタンを押しても何も表示されず、なぜ結果が出ないのか原因が全くわからなかったです。先輩に聞いたり、ChatGPTに間違っているところを解析してもらったりしましたが、間違っているところが見つからず、かなりの時間を費やしました。最終的には、一度最初から講義内容を見返し、今までのファイルをバックアップした上で書き直すことで解決しました。それまでは何度もコードを見直したり、console.logで値を確認したりして、本当に悩みました。 もう一つ苦労したのは、検索結果が表示された後に再度検索すると、前の結果が残ったまま新しい結果が追加されてしまう点でした。これに対しては、検索のたびに前の結果を消すようにprintDom関数の中で既存の表示エリアを削除する処理を加えることで対応しました。DOMを使ってWebページの要素を動的に変更できるという仕組みの理解にもつながり、授業で学んだことが実際の場面で活かせたと感じました。 工夫した点としては、情報の見せ方です。たくさんの項目がある中で、店名を大きめに表示し、アクセスや住所、予算など重要な情報を整ったレイアウトで表示できるようにしました。JavaScriptではinnerHTMLを使うことでHTMLの構造をまとめて書けるようにし、CSSではシンプルかつ読みやすいデザインを目指しました。特にジャンルごとの色分けや、ボックスでの表示などは、ユーザーが見やすくなるよう意識した部分です。 このプロジェクトを通じて、Webページはただ見た目を整えるだけでなく、「ユーザーが操作しやすい」「動作する」ように作ることがとても大切だということを実感しました。実際に自分で組んだコードでページが動き、外部のデータを取り込んで表示できるようになったときは、かなり達成感がありました。 今後は今回の経験を活かして、より複雑なページや機能の実装にも挑戦していきたいと思います。また自分の趣味で好きなようにWebページを作ってみたり、自分で作ったものが人に使われることを想像しながら、さらに実践的なスキルを身につけていきたいです。
